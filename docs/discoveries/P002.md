## Env Fallback Divergence

**Oracle:** `cd "${CLAUDE_PROJECT_DIR:-$(CDPATH= cd -- "$(dirname -- "${BASH_SOURCE[0]}")/.." && pwd)}"` — fallback = script's parent directory (always the project root, since the script lives in `scripts/`).

**Rust port:** `std::env::var("CLAUDE_PROJECT_DIR").unwrap_or_else(|_| current_dir())` — fallback = cwd at hook invocation time.

**Why accepted (CLAUDE.md Port doctrine constraint #7):** A Rust binary has no "script dir" concept. The oracle's fallback resolves the project root because the `.sh` file is inside `scripts/` — the binary cannot replicate this. Cwd is the closest safe fallback. In production, Claude Code always sets `CLAUDE_PROJECT_DIR` when firing PreToolUse hooks, making the fallback path a rare edge case. Ghi rõ trong Discovery — NOT a redesign.

## Isolation Strategy

Used `CLAUDE_PROJECT_DIR` env pointing to a per-test unique temp dir (`std::env::temp_dir().join(format!("claude-hooks-test-{name}-{pid}"))`) to create/omit `.sos-state/architect-active` without touching the real `.sos-state/` directory (which holds `worker-active` for the active session).

`tempfile` crate: NOT needed. `std::env::temp_dir()` + `std::fs::create_dir_all` + `std::fs::write` + `std::fs::remove_dir_all` (best-effort cleanup) was fully sufficient. No escalation required.

All 6 fire-test fixtures pass. No race condition possible because each test's `CLAUDE_PROJECT_DIR` is unique per process ID + test name.

## Pattern Representation

Chose plain `std::str` methods (`starts_with`, `contains`, `ends_with`) — no regex. Rationale: oracle uses POSIX `case` glob which maps 1:1 to these three primitives; no pattern requires regex alternation or lookahead. The `regex` crate is available in Cargo.toml but unnecessary here; simpler = fewer defects.

Port semantics respected:
- `__tests__/*` — prefix-only, NO `contains("/__tests__/")` added (oracle L59 has no segment variant).
- Build artifacts (`target/*`, etc.) — prefix-only, NO segment variant (oracle L61).
- `crates/*/src/*` — covered by `contains("/src/")` (already captured by segment rule) + `starts_with("src/")` (root-level). No special-case needed.
