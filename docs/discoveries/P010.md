### Tension 3 — why_blocked BOUNDED (RESOLVED, limitation noted)

`why_blocked` giữ route đơn: `Edit|Write` → `block_env_edit` only. Trong tarot deploy, `Write/Edit` fire CẢ `architect_guard` (allowlist check) LẪN `block_env_edit` (.env* check). `why_blocked` không mirror multi-hook firing. **Limitation:** dùng `why_blocked` với `Edit/Write` chỉ debug `block_env_edit`, không debug `architect_guard` Write/Edit block. Để debug architect_guard Write/Edit via MCP: gọi `architect_guard` tool trực tiếp với `tool_name="Write"/"Edit"`. Multi-hook why_blocked = finding/phiếu sau.

### Tension 4 — MultiEdit/NotebookEdit faithful (RESOLVED)

Dispatch chỉ `Read|Glob` + `Write|Edit`. `MultiEdit`/`NotebookEdit` → `_` default → ALLOW ở architect-guard (vẫn bị `block-env-edit` route riêng fire). Faithful tarot oracle case không match = ALLOW. KHÔNG thêm chúng vào allowlist branch.
