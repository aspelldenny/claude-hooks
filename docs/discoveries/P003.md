## Task 3 — env-fallback `CLAUDE_TOOL_INPUT` (CONSCIOUS PARITY GAP)

**Decision:** `CLAUDE_TOOL_INPUT` env-fallback (oracle `scripts/block-env-edit.sh` L16-20) is **intentionally NOT ported** in P003.

**Rationale:**

1. **Hook always receives stdin from Claude Code.** The PreToolUse hook spec sends JSON payload on stdin unconditionally. The oracle's env-fallback only activates when stdin is a tty (`[ ! -t 0 ]` fails), which never happens in production Claude Code execution.

2. **Shared harness change = Tầng 1 scope creep.** Adding env-fallback would require modifying `src/io.rs::read_payload()`, which is shared by P002 (`architect-guard`), P003 (`block-env-edit`), P004 (`block-unsafe-merge`), P005 (`session-banner`). That cross-hook API change is Tầng 1 and outside P003 scope.

3. **Consistency with P002.** `architect-guard` (P002) was ported without env-fallback under the same reasoning. Keeping the same decision avoids asymmetric behavior between hooks.

**Follow-on path:** If env-fallback is ever needed (e.g., for local debugging / manual pipe testing), create a dedicated phiếu targeting `src/io.rs` that covers all hooks simultaneously.

This gap is **not an oversight** — it is a documented deliberate deviation.

## Implementation Details

### basename approach

Used `path.rsplit('/').next().unwrap_or(&path)` (phiếu pseudocode recommendation). This gives the correct last segment for POSIX paths:
- `/a/b/.env` → `.env`
- `.env` → `.env`
- `x/.env` → `.env`

`std::path::Path::file_name()` would also work and be more robust for edge cases (trailing slashes), but `rsplit('/')` is sufficient for `file_path` / `notebook_path` values as sent by Claude Code Edit/Write/NotebookEdit tools. Tầng 2 decision, matches phiếu recommendation.

### regex vs string-match

Used `regex::Regex` as planned — dep confirmed present at `Cargo.toml:20`. Pattern `^\.env($|\.)` ported verbatim from oracle L44. The `unwrap()` on a constant literal pattern is safe (never fails to compile) and mirrors the phiếu's explicit guidance. No `LazyLock`/`once_cell` needed — hook process runs once, no recompile overhead.

The `($|\.)` alternation is the critical part: `.envrc` and `.environment` correctly do NOT match (character after `env` is neither `$` end-of-string nor `.`). Fixture case 3 (`p003_dot_envrc_allowed`) and case 10 (`p003_dot_environment_allowed`) verify this.

### notebook_path fallback

`tool_input.file_path.or(tool_input.notebook_path)` implements oracle L29-32 path fallback for `NotebookEdit` tool calls. Fixture case 8 (`p003_notebook_path_fallback_blocked`) verifies this path. The `pattern` field is deliberately not used (oracle does not check it for this hook).
